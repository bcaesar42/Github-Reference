How to host a webpage using GitHub Pages:

Step 1: Create a new repository.

In the upper right-hand corner of the page, click the little arrow next to the plus symbol.

Step 2: Name the new repository.

For convenience, you should also check the checkbox that says "Initialize this repository with a README". After naming the repo and checking the box, click the "Create repository" button.

Step 3: Upload your files to your new repository.

Using your computer's file explorer, drag and drop all the files for your webpage into the GitHub page. There should be an 'index.html' file, as well as two folders named 'css' and 'assets'.

Step 4: Go to the repository's settings menu.

To get to the settings menu, click on the settings tab near the top of the page.

Step 5: Scroll down to the section titled GitHub Pages.

These settings will be towards the bottom of the page, and will look like this:

Step 6: Select the Master branch.

In the 'Source' drop-down list, select 'master branch'. After you select 'master branch', the page will refresh. At this point, all you need to do is wait for GitHub to finish processing your request. Once your page is ready, the box at the top of the GitHub Pages section will turn green (the URL for your webpage will be inside this green box).
